================================================================================
KEY DOCUMENTATION POINTS ADDED
================================================================================

1. THREE RUN MODES EXPLAINED:
   - RunMode 0: Service/Console Mode (Default)
     * Shows console with logs
     * Best for debugging and troubleshooting
   
   - RunMode 1: Console + Tray Mode
     * Console with logs PLUS system tray icon
     * Best for development and monitoring
     * Quick access to configuration
   
   - RunMode 2: Tray Only Mode
     * System tray icon only, no console
     * Minimal interface
     * Best for production

2. HOW TO CONFIGURE:
   - Open configuration GUI (--tray or double-click tray icon)
   - Go to "Application Settings" tab
   - Select preferred "Default Run Mode" from dropdown
   - Click Save
   - Application now starts in chosen mode automatically

3. OVERRIDE BEHAVIOR:
   - Command line arguments still work
   - --tray forces tray-only mode (overrides config)
   - --console forces console+tray mode (overrides config)
   - No arguments = uses configured RunMode setting

4. CONFIGURATION FILE:
   - New ApplicationSettings section added
   - RunMode property (integer: 0, 1, or 2)
   - Persists with other configuration
   - Example format provided in all documentation

5. BACKWARD COMPATIBILITY:
   - Default RunMode = 0 (same behavior as before)
   - Existing configs automatically get ApplicationSettings
   - Command line arguments work exactly as before
   - No breaking changes
