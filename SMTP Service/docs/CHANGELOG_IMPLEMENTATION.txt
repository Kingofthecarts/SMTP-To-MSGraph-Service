================================================================================
MAINTAINING THE CHANGELOG
================================================================================

When to Add Entries:
✓ New features added
✓ Bugs fixed
✓ Performance improvements
✓ Documentation updates
✓ Configuration changes
✓ UI enhancements
✓ Security fixes

What to Include:
✓ Clear description of change
✓ User-facing impact
✓ Version number
✓ Date of change

What NOT to Include:
✗ Internal code refactoring (unless it affects users)
✗ Trivial changes (typo fixes in comments)
✗ In-progress features (only when complete)

Format Guidelines:
✓ Use present tense ("Added" not "Add")
✓ Start with verb (Added, Fixed, Improved, Updated)
✓ Be concise but clear
✓ Group similar changes together
✓ Most recent changes at the top

================================================================================
TECHNICAL DETAILS
================================================================================

File Reading:
- Uses AppDomain.CurrentDomain.BaseDirectory
- Path.Combine for cross-platform compatibility
- File.ReadAllText for simple read
- UTF-8 encoding (default)

Text Display:
- TextBox.Multiline = true
- TextBox.ReadOnly = true
- TextBox.ScrollBars = ScrollBars.Vertical
- Dock = DockStyle.Fill (responsive sizing)
- Select(0, 0) to scroll to top after load

Error Handling:
- Try-catch around file operations
- Meaningful error messages to users
- File path shown when file not found
- Exception details shown for unexpected errors
